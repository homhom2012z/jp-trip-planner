# Project Requirement Proposal (PRP)

## 1. Project Overview

**Goal**: Create a personalized Japan trip planning and location list website.
**Vibe**: "Japan Vacation Vibe" - Relaxing, clean, aesthetic, using Japanese design elements (vertical text where appropriate, soft colors, nature motifs).
**Target Platform**: Web (Responsive).

## 2. Core Features

### 2.1. City/Trip Pages

- **Main Cities**: Tokyo, Osaka.
- **1-Day Trips**: Kyoto, Uji, Kamakura.
- **Content**:
  - List of locations (Restaurants, Landmarks, Shopping Centers).
  - Map visualization of these locations.

### 2.2. Map Integration

- Display a map for each city/area.
- Markers for each location.
- **Requirement**: Display distance from the main transit point (e.g., "500m from Kyoto Station").
  - _Note_: This requires coordinate data (Latitude/Longitude) for both the destination and the transit point.

### 2.3. Location Details

- Name, Cuisine/Type.
- Price (JPY & THB).
- Photo (currently missing in CSV, will need a strategy).
- Distance info.

## 3. Technology Stack

- **Framework**: Next.js 16.1.1 (App Router).
- **Styling**: Tailwind CSS.
- **Language**: TypeScript.
- **Icons**: Lucide React or similar.
- **Map Library**: `react-map-gl` or `leaflet` (React Leaflet) for free OpenStreetMap usage, or Google Maps API (if API key provided). _Proposal: Use React Leaflet with OpenStreetMap for cost-efficiency unless Google Maps API key is available._

## 4. Data Strategy

### 4.1. CSV Handling

- The current CSVs contain: City, Name, Type, Price, Google Maps Link.
- **Missing Data**:
  - Coordinates (Lat/Lng) are needed for the map markers. The Google Maps link helps humans but isn't directly usable by a map library without extraction/geocoding.
  - 'Distance from Metro' column.
  - Images.

### 4.2. Proposed Solution for Missing Data

- **Coordinates**: We will attempt to extract coordinates from the Google Maps links if possible, or we may need to manually add Lat/Lng columns to the CSV for better accuracy.
- **Distance**: We can calculate the straight-line distance if we have the coordinates of the location and the main station.
- **Images**: We can use placeholder images generated by AI for the "vibe" or generic category images until real photos can be sourced.

## 5. UI/UX Design System

- **Color Palette**:
  - Primary: Sakura Pink / Matcha Green / Indigo Blue.
  - Background: Off-white / Washi paper texture.
- **Typography**: Clean serif or sans-serif combinations.
- **Components**:
  - `CityCard`: Entry point to city pages.
  - `LocationCard`: Details of a spot.
  - `MapComponent`: Interactive map.

## 6. Implementation Plan

1. **Setup**: Initialize Next.js project with Tailwind.
2. **Data Parsing**: Create a utility to read and parse the CSV files.
3. **Map Logic**: Implement a Map component that can take a list of locations.
4. **UI Implementation**: Build the "Japan Vibe" layout.
5. **Feature Completeness**: Add the distance calculation logic.

## Questions for User

- **Data**: Do you have the `Latitude` and `Longitude` for these spots, or should we try to extract them? (Extraction is tricky without an API).
- **Maps**: Do you have a Google Maps API Key? If not, we should use OpenStreetMap (Leaflet).
- **Images**: Since the CSV photo column is empty/formula-based, do you want us to generate placeholder assets?
